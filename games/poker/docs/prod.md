# 德州扑克 (Texas Hold'em) - Lob Games 功能需求设计文档

## 1. 项目概况
本项目是一个轻量级、无需后端的 P2P 网页德州扑克游戏。利用 PeerJS (基于 WebRTC) 实现玩家间的实时通信，主打“即开即玩”的社交竞技体验。

- **产品名称**: Lob 德州扑克
- **技术栈**: HTML5, Vanilla CSS, Vanilla JavaScript, PeerJS (WebRTC)
- **核心理念**: 无需注册、P2P 直连、极简美观、橙色调。
- **当前版本**: v1.2.0 (Enhanced Modular Version)

## 2. 核心功能模块

### 2.1 游戏大厅 (Lobby)
- **创建/加入房间**: 自动生成 4 位字符的房间号。
- **多玩家支持**: 支持 2-6 人同时对局（通过 Host 管理多路 P2P 连接）。
- **角色区分**:
  - **房主 (Host)**: 负责全局状态同步、倒计时启动、机器人添加/移除。
  - **玩家 (Player)**: 参与准备、对局及聊天。
- **状态同步**: 实时展示玩家准备状态。
- **动态入口**: 支持断线重连识别（基于 LocalStorage 存储的 UUID）。

### 2.2 P2P 通信 (Networking)
- **底层技术**: PeerJS + WebRTC 协议。
- **信令/穿透**: 配置了 Google、QQ、小米等多路公共 STUN 服务提升连接率。
- **协议同步**:
  - `ROOM_STATE`: 由房主定时广播，包含所有玩家筹码、底池、牌桌、阶段等全量信息。
  - `PLAYER_ACTION`: 玩家操作同步指令。
  - `ACTION_LOG`: 系统行为日志（如：XXX 已跟注）。
  - `CHAT`: 玩家实时文字交流。
  - `LEAVE`: 离场/掉线检测。

### 2.3 游戏逻辑引擎 (Game Engine)
- **盲注系统**: 自动处理庄家位 (Dealer)、小盲 (SB) 和大盲 (BB) 的轮替。
- **牌组管理**:
  - 52 张标准牌。
  - 实现 Fisher-Yates 洗牌。
- **完整的德州扑克阶段**:
  - `PREFLOP`, `FLOP`, `TURN`, `RIVER`, `SHOWDOWN`, `GAMEOVER`。
- **下注操作**: 支持 Check, Call, Raise, Fold, All-in。
- **智能逻辑**:
  - `handRank`: 牌型判定算法（支持 A-5 低顺子）。
  - `bestHandFrom7`: 从 7 张牌（2底+5公）中寻找最佳 5 张组合。
  - `compareRanks`: 牌力强弱对比。

### 2.4 用户界面 (UI/UX)
- **视觉风格**: 深度定制的“活力橙 (Lob Orange)”主题，融合毛玻璃背景、动态阴影。
- **交互动效**:
  - **粒子效果**: 筹码飞向底池的物理动效。
  - **发牌动画**: 带有延时的平滑发牌及其翻转效果。
  - **声效系统**: 点击、发牌、获胜 (SFX) 及大厅/对战背景音乐 (BGM)。
- **响应式桌台**: 核心玩家位置置底展示，对手环绕式排列。
- **实时看板**: 动态显示本轮下注、总底池、当前阶段及系统日志。

### 2.5 机器人 (AI)
- **行为逻辑**: 基于 `estimateHandStrength` 手牌强度评分，结合随机因子模拟真实决策。
- **托管模式**: 房主可随时添加 Bot 进行对战。

## 3. 技术架构 (Modular Type="module")
- **`js/app.js`**: 应用控制中心 (Controller)。处理路由、消息分发、生命周期。
- **`js/poker.js`**: 领域驱动逻辑 (Engine)。纯逻辑层，不依赖 DOM。
- **`js/net.js`**: 通信层 (Infrastructure)。封装 P2P 连接与数据透传。
- **`js/ui.js`**: 表现层 (View)。负责 DOM 更新、动画触发、媒体控制。

## 4. 技术现状分析 (Status v1.2.0)
- **稳定性**: 已引入 UUID 方案支持网页刷新后的身份一致性及重连逻辑。
- **性能**: 核心判定引擎经过优化，大频率广播状态时无明显下行压力。
- **体验**: 已完成核心动效（发牌/筹码/转场）与音效系统的集成。
- **已知不足**: WebRTC 穿透率受 PeerJS 公共信令服务器负载影响，极端 NAT 下连接较慢。
