# 德州扑克 (Texas Hold'em) - Lob Games 功能需求设计文档

## 1. 项目概况
本项目是一个轻量级、无需后端的 P2P 网页德州扑克游戏。利用 PeerJS (基于 WebRTC) 实现玩家间的实时通信，主打“即开即玩”的社交竞技体验。

- **产品名称**: Lob 德州扑克
- **技术栈**: HTML5, Vanilla CSS, Vanilla JavaScript, PeerJS (WebRTC)
- **核心理念**: 无需注册、P2P 直连、极简美观、橙色调。

## 2. 核心功能模块

### 2.1 游戏大厅 (Lobby)
- **创建房间**: 自动生成 4 位字符的唯一房间号。
- **加入房间**: 支持输入 4 位房间号加入已有对局。
- **角色区分**:
  - **房主 (Host)**: 负责管理房间状态、启动倒计时、添加/删除 Bot。
  - **玩家 (Player)**: 参与准备、对局。
- **状态同步**: 实时显示各玩家准备状态（✓ 已准备 / 等待中）。
- **准备/开始**: 所有玩家准备就绪后，房主触发 3 秒倒计时开启游戏。

### 2.2 P2P 通信 (Networking)
- **底层技术**: 采用 WebRTC 协议，通过 PeerJS 的 ID 系统建立 P2P 连接。
- **信令服务**: 使用默认或备用 STUN 服务进行网络穿透。
- **协议同步**:
  - `ROOM_STATE`: 全局房间状态同步（同步所有玩家筹码、底池、手牌等）。
  - `ACTION`: 玩家操作同步（跟注、加注、弃牌等）。
  - `CHAT`: 玩家实时文字交流。
  - `LEAVE`: 玩家离场检测。

### 2.3 游戏逻辑引擎 (Game Engine)
- **盲注系统**: 包含小盲 (Small Blind) 和大盲 (Big Blind) 逻辑，并在玩家间轮替。
- **牌组管理**:
  - 52 张标准扑克，包含花色 (♠, ♥, ♦, ♣) 和点数 (2-A)。
  - Fisher-Yates 洗牌算法。
- **牌局阶段**:
  1. `PREFLOP`: 各发 2 张底牌，开始首轮下注。
  2. `FLOP`: 发 3 张公共牌。
  3. `TURN`: 发第 4 张公共牌。
  4. `RIVER`: 发第 5 张公共牌。
  5. `SHOWDOWN`: 摊牌环节，结算胜负。
- **下注操作**:
  - `Check` (让牌)
  - `Call` (跟注)
  - `Raise` (加注)
  - `Fold` (弃牌)
  - `All-in` (全下)
- **胜负判定**: 实现完整的 7 选 5 最佳牌组算法（高牌到同花顺）。

### 2.4 用户界面 (UI/UX)
- **视觉风格**: 以“暖色/橙色 (Orange)”为基调的高级质感设计，采用毛玻璃效果 (Backdrop Blur)、阴影和微动效。
- **游戏桌布局**: 模拟真实牌台，位于中央，各玩家信息环绕分布。
- **操作栏**: 动态按钮布局，仅在当前操作玩家显示。集成加注面板及金额输入。
- **实时聊天**: 底部叠层聊天，支持系统消息（如：XX 加入房间、XX 弃牌）。
- **结果弹窗**: 局末展示获胜者姓名、最佳牌组及手牌缩略。

### 2.5 机器人 (AI)
- **基础行为**: 基于手牌强度估值实现。
- **逻辑**: 计算手牌价值 (Score)，结合当前底池投入情况，模拟真实玩家的弃牌、跟注或加注逻辑。
- **适用场景**: 补位、单人练习模式。

## 3. 技术架构设计 (ES Modules)
项目采用现代浏览器原生模块化架构 (Type="module")：
- **`index.html`**: 主页面入口与 UI 模板。
- **`js/poker.js`**: 核心逻辑引擎。包含牌组生成、手牌判定算法、庄闲盲注逻辑及 AI 决策模型。
- **`js/net.js`**: P2P 通信模块。封装 PeerJS 逻辑，处理 WebRTC 数据传输与信令同步。
- **`js/ui.js`**: UI 渲染层。负责 DOM 的动态更新、牌桌布局渲染、操作栏显隐及倒计时处理。
- **`js/app.js`**: 应用控制台。连接各模块，负责状态管理、消息路由及全局生命周期控制。
- **`css/style.css`**: 全局样式表。支持响应式布局与主题定义。

## 4. 技术现状分析 (Refactored)
- **模块化状态 [√ 已完成]**: 对原有的 1400+ 行单文件进行了彻底重构。现在各逻辑层清晰解耦，具备良好的可维护性。
- **P2P 稳定性**: 目前依赖 PeerJS 主服及公共 STUN。建议后续引入更多备选 TURN 服务器以提升成功率。
- **容错机制**: 目前支持基础的断线检测。需进一步完善“房主自动迁移 (Auto Host Migration)”逻辑。
- **单元测试**: 核心判定引擎 (`handRank`) 仍需增加自动化测试覆盖，以确保在复杂牌型下的博弈公正。
