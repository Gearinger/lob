## Session State Corruption (2026-02-11)

### Symptom
- Error: `LLM request rejected: invalid params, tool result's tool id(call_function_xxx) not found (2013)`
- Occurs after interrupted tool calls or Gateway instability

### Why It Happens
- LLM provider (MiniMax) caches a tool call ID
- Session context becomes "polluted" with mismatched IDs
- Restarting Gateway alone doesn't fully clear the state

### Solution
- **Most Effective:** Start a new session with `/new`
- Less effective: `openclaw gateway restart` (sometimes helps, sometimes not)

### Lesson Learned
| Error Type | Best Solution |
|------------|----------------|
| Minor tool errors | Retry / modify |
| Gateway issues | Restart Gateway |
| Invalid tool ID | `/new` immediately |

**When you see "invalid params, tool id not found" → Don't waste time, just `/new`**

## Memory Architecture

### What Persists vs What Resets

| Type | Persists? | Storage |
|------|-----------|---------|
| Conversation history | ❌ No | Session memory (lost on /new) |
| Memory files | ✅ Yes | MEMORY.md, memory/*.md |
| Project files | ✅ Yes | workspace/, project/ |

### Recovery Flow After /new
1. New session starts (no conversation history)
2. OpenClaw auto-reads MEMORY.md + memory/*.md
3. I restore long-term context
4. Answer is based on persisted memory, not transient chat
