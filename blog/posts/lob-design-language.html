规范文档 · Design System

Lob Design Language v1.6

发布日期：2026-05-01 · 状态：正式规范

设计目标：简洁、明确、优雅、协调。动效只出现在用户需要知道"发生了什么"的时刻。

一、设计原则

所有设计决策以此为准绳。遇冲突，以此判断。

核心理念

简洁 Simplicity
删到删不动为止。每个元素必须回答"它做什么"，否则删掉。
明确 Clarity
层级清晰、目的清晰、交互清晰。没有歧义，用户一看就知道这是什么、能做什么。
优雅 Refinement
恰到好处，不用力过猛。细节精致但不刻意，克制是最高级的讲究。
协调 Consistency
全站统一语言。切换页面不感觉跳，像出自同一种审美和思路。

执行准则

语义化颜色
颜色有含义，不是装饰。强调色用于可交互元素，背景色区分层级。待完善
内容优先
设计退后，内容前进。界面元素最小化，内容最大化。
触摸目标 ≥ 44pt
所有可点击元素的最小尺寸为 44×44pt，确保可操作性。
间距节奏
基于 8px Grid，内容之间有呼吸。克制留白，不过度铺张。
动效原则：动效只服务于反馈，不服务于装饰。用户点击 → 按钮回应；悬停 → 元素回应。轻、准、快。没有目的的动效一律删除。

2.4 强调色（固定）

accent 色固定为以下值，不再随时间变化。

变量名色值说明
--accent-primary#67e8f9主强调色，天蓝（固定）
--accent-secondary#a78bfa次强调色，薰衣草紫（固定）
--accent-glowrgba(103, 232, 249, 0.35)光晕效果

2.5 禁止使用的颜色

以下颜色在 Phase 1 后的任何页面中禁止出现：

  • #FFF5EE / #FFFAF8 — 旧版暖白背景
  • #3D2A35 — 旧版深色文字
  • #E88A3D / #FFB07A — 旧版橙色 accent
  • #FF8FA3 / #A8D8FF — 旧版渐变
  • 任何暖色系：rgba(255, 143, ...) / rgba(200, 140, ...) / rgba(168, 216, ...)

2.6 语义化颜色（待完善）

待完善 当前颜色变量使用功能性命名（如 --bg-deep、--text-primary），符合基本规范但尚未达到 Apple HIG 的语义化标准。目标命名如 --color-surface、--color-on-surface 等，应在后续迭代中重构。

三、字体系统

使用系统字体（-apple-system, "PingFang SC", "Helvetica Neue", Arial, sans-serif），参考 Apple 的字体策略：干净、可预测、加载快。

已弃用：Ma Shan Zheng（毛笔字体）和 ZCOOL XiaoWei（站酷犀利体）已弃用，装饰性太强，与暗色简洁风格冲突。

字号层级

元素字号字重其他
文章大标题 (h1)clamp(2rem, 6vw, 2.8rem)700letter-spacing: -0.02em
段落标题 (h2)1.1rem600—
小标题 (h3)0.92rem600—
正文0.88rem—line-height: 1.8
副文本/描述0.82rem—color: --text-secondary
辅助信息0.72rem—color: --text-muted
标签0.68rem—letter-spacing: 0.05em

正文字号 0.88rem，比行业常见（16px）偏小，理由：暗色背景下字稍小更显精致，同时增加留白感。

四、间距系统

基于 8px Grid。间距要慷慨，内容之间有呼吸。

名称值用途

--space-sm12px紧凑元素间距
--space-md20px标准间距
--space-lg24px页面水平 padding
--space-xl40px区块间距
--space-2xl64px大区块间距

五、动效规范

核心原则：动效只出现在用户需要反馈的时刻。轻（opacity 变化小）、准（target 精确）、快（0.2–0.3s）。没有目的的动效一律删除。

5.1 禁止使用的动效

禁止 — 粒子背景：无功能性价值，增加视觉噪音，所有页面不使用。

禁止 — 页面切换过渡动画（渐变、滑动），加载即呈现，无过渡。

禁止 — scale 放大、shadow 剧烈扩散、过长的过渡时间（> 0.35s）。

5.2 允许的动效

  • 卡片悬停：translateY(-4px) + 边框颜色变亮，0.2s ease
  • 按钮悬停：背景色加深 + 边框变亮，0.2s ease
  • 输入框聚焦：边框色变为 accent，0.2s ease

5.3 加载状态

骨架屏（Skeleton）优于 spinner。骨架屏使用 --bg-surface 色块，动画脉冲。

.skeleton { background: var(--bg-surface); border-radius: var(--radius-sm); animation: skeleton-pulse 1.5s ease-in-out infinite; } @keyframes skeleton-pulse { 0%, 100% { opacity: 1; } 50% { opacity: 0.4; } }

六、圆角系统

基于 4px 递进，层级分明。

名称值用途
--radius-sm6px标签内边角、小按钮
--radius-md12px卡片、输入框、弹出层
--radius-lg16px大卡片、模态框
--radius-pill9999px胶囊按钮、标签圆角

规则：大元素用大圆角，小元素用小圆角。同一个元素不要混用多个圆角值。

七、阴影系统

暗色模式下阴影效果有限，尽量使用边框而非阴影区分层级。如需阴影，使用以下规范：

名称值用途
--shadow-sm0 1px 3px rgba(0,0,0,0.4)轻微浮起（hover态）
--shadow-md0 4px 12px rgba(0,0,0,0.5)卡片悬浮态
--shadow-modal0 8px 40px rgba(0,0,0,0.7)模态框（慎用）
原则：阴影仅在 hover/focus 态使用，正常态用边框。暗色模式下阴影不如亮色明显，过度使用显得脏。

八、Z-Index 层级

避免 z-index 冲突，统一命名空间。

变量名值用途
--z-base0页面内容
--z-header100固定 Header
--z-overlay200弹出层/模态框背景
--z-modal300模态框内容
--z-toast400Toast 提示

九、组件规范

所有组件代码示例统一使用 CSS 变量，不使用硬编码色值。

9.1 玻璃态 Header

position: fixed; top: 0; left: 0; right: 0; z-index: var(--z-header, 100); backdrop-filter: blur(20px) saturate(1.8); -webkit-backdrop-filter: blur(20px) saturate(1.8); background: rgba(0, 0, 0, 0.45); border-bottom: 1px solid var(--border);

9.2 卡片

background: var(--bg-card); border: 1px solid var(--border); border-radius: var(--radius-md); padding: 24px; transition: border-color 0.2s ease, transform 0.2s ease; /* 悬停 */ border-color: rgba(103, 232, 249, 0.33); transform: translateY(-4px);

使用边框色变化代替 box-shadow，悬停时边框由半透明变为微亮，视觉更干净。

9.3 标签

半透明填充色使用 rgba 硬编码（CSS 变量不支持 opacity 运算），实际使用时可用预定义变量 --accent-fill 替代。

font-size: 0.68rem; padding: 3px 10px; border-radius: var(--radius-pill); background: rgba(103, 232, 249, 0.1); border: 1px solid rgba(103, 232, 249, 0.2); color: var(--accent-primary); letter-spacing: 0.05em;

9.4 按钮

同上，半透明填充色使用 rgba 硬编码。

padding: 8px 20px; border-radius: var(--radius-pill); font-size: 0.82rem; background: rgba(103, 232, 249, 0.1); border: 1px solid rgba(103, 232, 249, 0.2); color: var(--accent-primary); transition: background 0.2s ease, border-color 0.2s ease; /* 悬停 */ background: rgba(103, 232, 249, 0.2); border-color: var(--accent-primary);

9.5 分隔线

height: 1px; background: linear-gradient(90deg, transparent, var(--border), transparent);

十、状态设计

每种状态都需要对应的视觉表现。

10.1 加载状态（Loading）

骨架屏（Skeleton）优于 spinner。骨架屏使用 --bg-surface 色块，动画脉冲。

.skeleton { background: var(--bg-surface); border-radius: var(--radius-sm); animation: skeleton-pulse 1.5s ease-in-out infinite; } @keyframes skeleton-pulse { 0%, 100% { opacity: 1; } 50% { opacity: 0.4; } }

10.2 空状态（Empty）

居中显示，提供引导性插图/图标 + 简短文案 + 可选操作按钮。

.empty-state { display: flex; flex-direction: column; align-items: center; justify-content: center; gap: 16px; padding: 80px 24px; text-align: center; } .empty-state-icon { font-size: 3rem; opacity: 0.3; } .empty-state-title { font-size: 1rem; font-weight: 600; color: var(--text-primary); } .empty-state-desc { font-size: 0.82rem; color: var(--text-muted); }

10.3 错误状态（Error）

红色边框 + 红色提示文字，简洁明确，不使用弹窗打断用户。

全局错误使用 Toast 提示（--z-toast），位于页面顶部中央，3秒后自动消失。

10.4 禁用状态（Disabled）

opacity 降至 0.4，cursor 改为 not-allowed，不使用灰色背景（灰色在暗色模式下容易脏）。

:disabled, .disabled { opacity: 0.4; cursor: not-allowed; pointer-events: none; }

十一、图片规范

  • 圆角：所有图片统一 border-radius: var(--radius-md)
  • 填充方式：object-fit: cover，确保比例一致
  • Loading：占位图使用 --bg-surface 灰色块
  • 懒加载：所有非首屏图片使用 loading="lazy"
  • 格式：优先 WebP，次选 PNG/JPG。动画用 GIF 或 WebP。
  • 尺寸：明确 width/height 属性，避免 CLS（Cumulative Layout Shift）

十二、图标规范

  • 尺寸：导航/工具栏图标 20-24px，装饰图标 16px，卡片图标 40px（container）
  • 颜色：统一使用 color: var(--accent-primary) 或 currentColor，随主题色变化
  • 来源：使用 Lucide Icons（MIT 协议），本地存储 SVG 文件
  • 内联：图标 SVG 直接内联到 HTML，避免额外 HTTP 请求
  • 禁止：不使用 emoji 作为功能性图标，emoji 仅用于表情/游戏等上下文

十三、响应式断点

断点宽度调整
桌面> 640px默认布局
移动≤ 640px单列布局，padding 减半，隐藏次要元素

十四、文件规范

  • colors.css：所有 CSS 变量定义，所有页面引用此文件
  • 每个页面 <link rel="stylesheet" href="../colors.css">
  • 每个页面的 JS：仅保留必要的功能性脚本，不添加装饰性动效
  • 颜色值：统一使用 CSS 变量，禁止硬编码色值（组件代码示例除外，但应以使用变量为佳）
