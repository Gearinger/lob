# 躲弹幕 — 产品需求文档

## 1. Concept & Vision

一款极简主义弹幕生存游戏。玩家操控红色方块在黑色空间中躲避从四面八方涌来的弹幕，同时自动反击消灭子弹。一局几十秒到几分钟，死亡即重来——简单到无脑，上瘾到停不下来。

**核心情绪：** 紧张、专注、心流。活着就是胜利。

---

## 2. Design Language

### 视觉风格
- **整体：** 深色极简 + 霓虹发光感，像太空里的激光表演
- **背景：** `#0a0a0f` 纯黑，微弱网格线增加空间感

### 色彩
| 元素 | 颜色 |
|------|------|
| 背景 | `#0a0a0f` |
| 玩家 | `#ff4444` 红色方块 + 红色辉光 |
| 玩家子弹 | `#4af` 蓝色发光圆点 |
| 敌弹 | 暖色调 HSL，随波次变亮：`hsl(30+wave*10, 80%, 50+wave*3%)` |
| 粒子爆发 | 继承子弹颜色 |
| Power-up | Emoji：`❤` `⚡` `💨` `🔥` |

### 字体
- 系统字体 `system-ui`，分数显示大号粗体

### 动效
- 玩家：呼吸式脉动 `scale: 1 ± 0.05`
- 玩家拖尾：渐隐的半透明方块轨迹
- 弹幕：辉光 `shadowBlur: 10`
- 碰撞：粒子爆发
- Power-up：脉动漂浮

---

## 3. Layout & Structure

### 屏幕
- **全屏 Canvas** — 固定填满视口，无滚动
- **顶部 HUD** — 左：波次 | 右：分数（大字）
- **底部提示** — 操控方式，游戏中隐藏

### 状态
1. **Menu** — 标题 + 说明 + 开始按钮
2. **Playing** — 游戏主画面
3. **Game Over** — 显示最终分数 + 最高分 + 重来按钮

---

## 4. Features & Interactions

### 操控
- **键盘：** WASD / 方向键，diagonal 移动归一化
- **触屏：** 单指滑动，delta 映射方向
- **触屏：** 手指抬起后方向归零

### 游戏循环
1. 玩家出生在屏幕中央，1.5 秒无敌/无弹幕
2. 敌弹从屏幕边缘向玩家位置飞来（波次递增）
3. 玩家移动躲避，或用子弹击毁敌弹
4. 敌弹碰到玩家 → 扣 1 血，60 帧无敌
5. 血量归零 → 游戏结束

### 弹幕系统
- **生成：** 每波 N 个子弹从屏幕边缘飞向玩家当前位置
- **波次递增：** 每 300 帧（5 秒）波次 +1
- **难度曲线：**
  - 敌弹数量：`3 + floor(wave * 1.5)`
  - 敌弹速度：`1.5 + wave * 0.2 + random(0~1)`
  - 生成间隔：`max(50, 90 - wave * 5)` 帧

### 自动射击
- 每 30 帧（0.5 秒）射击一次
- 有 power-up 时多向散射
- 无 power-up 时 4 向射击（上下左右）

### Power-up 系统
| 类型 | Emoji | 效果 |
|------|-------|------|
| Heal | ❤ | 血量 +1，上限 5 |
| Power | ⚡ | 获得 8 向散射炮 |
| Shield | 💨 | 120 帧（2 秒）无敌 |
| Multishot | 🔥 | 获得 4 向射击炮（叠加） |

- 掉落概率：每波 15%
- 存在时间：600 帧（10 秒）
- 拾取：碰撞即生效

### 得分
- 每帧 +1（存活奖励）
- 子弹击毁敌弹 +5
- 最高分本地 `localStorage` 持久化

### 碰撞
- **玩家 vs 敌弹** → 扣血 + 无敌帧 + 粒子爆发
- **玩家子弹 vs 敌弹** → 双方湮灭 + 粒子爆发 + 得分

---

## 5. Component Inventory

### 玩家方块
- 14×14 红色方块，红色辉光
- 呼吸脉动动画
- 受伤时闪烁（每 4 帧 toggle 可见）
- 血量显示为方块下方小绿点

### 敌弹
- 圆形，半径 4px，有辉光
- 颜色随波次变化

### 玩家子弹
- 圆形，半径 5px，蓝色
- 速度 7 px/帧

### Power-up
- Emoji 漂浮，有脉动效果
- 靠近玩家时自动拾取

### 粒子
- 碰撞时爆发，渐隐缩小消失

### HUD
- 左上：波次文字（小号）
- 右上：分数（大号粗体 + 红色辉光）

### 菜单覆盖层
- 半透明黑底
- 标题 + 说明文字 + 开始按钮

---

## 6. Technical Approach

- **单文件 HTML** — 零依赖，`requestAnimationFrame` 驱动
- **Canvas 2D** — 纯渲染，无 DOM 操作
- **状态机** — `menu | playing | gameover`
- **持久化** — `localStorage` 存最高分
- **响应式** — `resize` 事件重算 W/H/cx/cy
